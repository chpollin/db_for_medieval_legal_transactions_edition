External Evaluation and Quality Assessment¶

The external evaluation (Chair of Digital Humanities, University of Graz, 29 Dec 2025) assesses the data resource and web application of the project "Stadt und Gemeinschaft" against RIDE catalogues, JODH guidelines, and other DH standards. The overall verdict is positive with conditions: release of the repository and web application is recommended, subject to specific requirements. The repo analysis confirms most findings, adds quantitative data, and identifies additional issues not covered by the evaluation.

1. Overall Assessment¶

The scholarly substance is solid; the technical implementation is not. This is not unusual for a humanities research project -- the priority was on content, not software engineering. The consequence is a repository that works as long as the original editors maintain it, but cannot be handed over, published, or reused long-term without revision.

The identified deficits are fixable and follow a clear pattern: missing standardisation (encoding, delimiter, documentation format), missing automation (tests, CI/CD, reproducible pipeline), and missing cleanup (ballast files, duplicates, security issue). None of these require a rebuild -- this is consolidation and modernisation work.

2. Evaluation vs. Repo Findings¶

Evaluation vs. Repo Findings (Summary)¶

All 13 confirmed findings have been addressed: licence (Phase A), CRediT roles (Phase A), quality controls (395 tests, CI/CD via GitHub Actions), DTD replaced by RelaxNG (Phase B), newpersonList deleted, XSLT replaced by Python (Phase D+E), README corrected. Scholarly substance confirmed: factoid model, annotation quality, source-proximate approach.

4 evaluation findings quantified by repo analysis: 326 DOCTYPEs (fixed B.2), 3,530 files + 7,102 normalisation entries (exact counts). 7 additional findings discovered: credentials (fixed A), UTF-7 encoding, empty tables, place underrepresentation, Saxon JAR (resolved D+E), ballast files (deleted), duplicates (fixed A).

9 web frontend findings from evaluation: time filter, autocomplete, REST API, persistent IDs, PHP versions, URI errors, ODD, XPointer (fixed), triggerstring docs. XPointer conformance now done (48,421 attributes, Session 24).

Appendix: Evaluation Original Text (Abridged)¶

Evaluator: Chair of Digital Humanities, University of Graz. Date: 29 Dec 2025. Subject: Data resource and web application of the project "Stadt und Gemeinschaft". Basis: Git repo (commit 99ce004e45, Nov 2025), web application code (Nextcloud), web application (Dec 2025), project context sketch.

Key Decisions¶

Question 1 -- Release GitHub repository: Yes, with conditions. XML annotations can be made public in their current state in the short term. No dependency on the interface.

Question 2 -- Release web application: Yes, with conditions. The use of the data for the specific project context is on good track with the web interface under development.

Conditions for GitHub Release¶

Short-term (prerequisite for publication): 1. Clear data licensing (e.g. licence document) 2. Add ODD file (element definitions + annotation description from DB_presentation as introductory text) 3. Fix validation errors in the data 4. Adapt @ref and @corresp to XPointer rules 5. Explain individualisation procedure in the indices 6. Consolidate encoding guidelines from various files in the ODD (e.g. resolution of editor abbreviations from /normalisation_lists/editors/Name_of_editors.csv)

Medium-term: - Plan resources for fixing further errors mentioned above - Document transformation scripts in more detail - Develop quality assurance measures for further data collection

Long-term: - Convert and publish relational data in RDF - Project onto CIDOC-CRM-compatible schema - Migrate annotations to BeNASch schema

Scholarly Reuse Potential (Evaluation Assessment)¶

Scope: 16,055 individualised persons, 32,034 person mentions, 12,554 events. Prosopographically, socially, and economically highly relevant source collection. Economic and legal actions of Viennese citizens at a density that is relevant even in inter-city comparison. Potential NLP test and training material. Systematic data matching with GND/WikiData via OpenRefine recommended.
